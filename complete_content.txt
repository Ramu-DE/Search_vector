What is a vector database?
STORE AND RETRIEVE VECTOR EMBEDDINGS

•

A vector datastore or database enables you to store and query high dimensional vectors at scale, with
efficient nearest neighbor query algorithms and appropriate indexes to improve data retrieval

•

Enables updating and deleting vector embeddings as information changes e.g. product description
updates

•

Vector databases offer ability to filter information based on similarity and metadata e.g. embeddings of
product description field, numeric fields for product price

Putting It Together —Decision Framework
Step 1: Define your latency SLA (P99 < 200ms? < 500ms? < 2s?)
Step 2: Set minimum acceptable recall (Recall@10 > 0.85? > 0.95?)
Step 3: Cost becomes the optimization variable
E-commerce

Internal Knowledge Base

Low latency +High recall → Invest in infra
In-memory HNSW, pre-filtering, aggressive caching

Moderate latency OK → Optimize cost
Disk-based vectors, sparse encoding, OCU auto-scale

Legal / Compliance

Chatbot / RAG

Maximum recall required → Accept higher cost +latency
Exact k-NN, hybrid search, reranking pipeline

Balance all three → Hybrid search +caching
Domain fine-tuned model, quantized vectors, tiered storage

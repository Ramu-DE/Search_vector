1.1 What is a Vector?
─────────────────────────────────────────────────────────────────────

A vector is a list of numbers representing something in n-dimensional space.

Real-World Analogy: House Properties
┌────────────────────────────────────────────────────────────────┐
│ House 1: [5 bedrooms, 4 baths, 854 sqm, $1.1M]               │
│ House 2: [4 bedrooms, 3 baths, 335 sqm, $530K]               │
│                                                                 │
│ Vector Space:                                                   │
│     Price                                                       │
│       │                                                         │
│   2M  │     ● House 1                                          │
│       │                                                         │
│   1M  │                                                         │
│       │                ● House 2                                │
│    0  └──────────────────────────────────── Bedrooms           │
│           0   2   4   6   8                                     │
│                                                                 │
│ Distance → Similarity                                           │
│ Closer vectors = More similar houses                            │
└─────────────────────────────────────────────────────────────────┘

Text as Vectors (Embeddings):
┌─────────────────────────────────────────────────────────────────┐
│ "The cat sat on the mat"                                        │
│         ↓  (Embedding Model)                                    │
│ [0.234, -0.567, 0.891, 0.123, ..., -0.345]  (768 dimensions)  │
│                                                                 │
│ Each dimension captures some semantic feature:                  │
│ - Dimension 42: "animal-ness"                                   │
│ - Dimension 156: "action-ness"                                  │
│ - Dimension 234: "location-ness"                                │
│ etc.                                                            │
└─────────────────────────────────────────────────────────────────┘

1.2 Embedding Generation Process
─────────────────────────────────────────────────────────────────────

Step 1: Tokenization
Input: "The cat sat"
  ↓
Tokens: ["The", "cat", "sat"]
  ↓
Token IDs: [101, 2054, 4938]

Step 2: Token Embeddings
Each token → Vector
"The" → [0.1, -0.2, 0.5, ...]  (768d)
"cat" → [-0.2, 0.5, 0.1, ...]  (768d)
"sat" → [0.3, 0.1, -0.4, ...]  (768d)

Step 3: Contextual Processing (Transformer)
Self-Attention: Each token looks at others

"The [cat] sat"
  cat attends to: The (subject) + sat (action)
  → Meaning: domestic animal performing action

Step 4: Pooling
Average all token embeddings → Single sentence vector
Final: [0.067, 0.133, 0.067, ...]  (768d)

Complete Flow:
Text → Tokenize → Token Embeds → Transformer → Pool → Vector

2.1 What is a Vector Database?
─────────────────────────────────────────────────────────────────────

A specialized database optimized for:
1. Storing high-dimensional vectors
2. Fast similarity search (k-NN)
3. Combining with traditional filters

OpenSearch as Vector Database:
┌─────────────────────────────────────────────────────────────────┐
│ Document:                                                        │
│ {                                                                │
│   "title": "The Matrix",                     ← Text field       │
│   "genre": ["Sci-Fi", "Action"],             ← Keyword field    │
│   "rating": 8.7,                              ← Numeric field    │
│   "title_vector": [0.23, -0.56, ..., 0.89]  ← Vector field     │
│ }                                                                │
│                                                                  │
│ Storage:                                                         │
│ ┌───────────────────┐   ┌──────────────────┐                   │
│ │ Inverted Index    │   │ Vector Index     │                   │
│ │ (Lucene)          │   │ (HNSW)           │                   │
│ ├───────────────────┤   ├──────────────────┤                   │
│ │ "matrix" → [1]    │   │ Layer 2: A       │                   │
│ │ "sci-fi" → [1,5]  │   │ Layer 1: A─B     │                   │
│ │ "action" → [1,3]  │   │ Layer 0: A─B─C   │                   │
│ └───────────────────┘   └──────────────────┘                   │
│                                                                  │
│ Query Types:                                                     │
│ 1. Keyword: Use inverted index (fast exact match)              │
│ 2. Semantic: Use vector index (similarity search)              │
│ 3. Hybrid: Combine both! (best results)                        │
└──────────────────────────────────────────────────────────────────┘

2.2 HNSW Algorithm (Hierarchical Navigable Small Worlds)
─────────────────────────────────────────────────────────────────────

Key Idea: Multi-layer graph like highway system

Highway System Analogy:
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│ Interstate (Layer 2):  A ════════════════════ B                 │
│                        ║                      ║                 │
│                        ║                      ║                 │
│ State Roads (Layer 1): A ════ C ════ D ════ B                  │
│                        ║      ║      ║      ║                  │
│                        ║      ║      ║      ║                  │
│ Local (Layer 0):       A ─ C ─ E ─ D ─ F ─ B                   │
│                                                                  │
│ Search from A to B:                                             │
│ 1. Start at layer 2 (fast, long jumps)                         │
│ 2. Move down to layer 1 (more options)                         │
│ 3. Arrive at layer 0 (precise navigation)                      │
│                                                                  │
│ Result: Log(N) time complexity instead of O(N)!                 │
└──────────────────────────────────────────────────────────────────┘

HNSW Search Process:
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│ Query: [0.5, 0.3, 0.8, ...]                                     │
│                                                                  │
│ Step 1: Start at entry point (top layer)                        │
│   Layer 2:  ● (entry)                                           │
│             │                                                    │
│             ↓                                                    │
│                                                                  │
│ Step 2: Greedy search - follow closest neighbors                │
│   Layer 1:  ● ─→ ● ─→ ● (getting closer)                       │
│             │    │    │                                         │
│             ↓    ↓    ↓                                         │
│                                                                  │
│ Step 3: Bottom layer - beam search with ef_search               │
│   Layer 0:  ● ─ ● ─ ● ─ ● ─ [Target]                          │
│             Explore ef_search=100 candidates                     │
│             Return top K=10 closest                              │
│                                                                  │
│ Complexity: O(log N) vs O(N) for brute force                    │
│ Speed: 10-100x faster!                                          │
└──────────────────────────────────────────────────────────────────┘

SECTION 3: DISTANCE METRICS
═══════════════════════════════════════════════════════════════════════

3.1 Cosine Similarity (Most Common for Text)
─────────────────────────────────────────────────────────────────────

Formula: similarity = (A · B) / (||A|| × ||B||)

Visual Representation:
┌─────────────────────────────────────────────────────────────────┐
│        B [3,4]                                                   │
│       ╱│                                                         │
│      ╱ │                                                         │
│     ╱  │  Cosine measures ANGLE                                 │
│    ╱ θ │  (not distance)                                        │
│   ╱____│                                                         │
│  A [4,2]                                                         │
│                                                                  │
│ Same direction → High similarity (even if different magnitude)   │
│                                                                  │
│ Examples:                                                        │
│ [1, 1] vs [2, 2]  →  similarity = 1.0 (perfect!)               │
│ [1, 0] vs [0, 1]  →  similarity = 0.0 (perpendicular)          │
│ [1, 1] vs [-1,-1] →  similarity = -1.0 (opposite)              │
│                                                                  │
│ Range: [-1, 1]                                                   │
│ Best for: Text similarity, document retrieval                    │
└──────────────────────────────────────────────────────────────────┘

3.2 Euclidean Distance (L2)
─────────────────────────────────────────────────────────────────────

Formula: distance = √(Σ(ai - bi)²)

Visual Representation:
┌─────────────────────────────────────────────────────────────────┐
│        B [6,8]                                                   │
│        ●                                                         │
│       ╱│                                                         │
│      ╱ │                                                         │
│     ╱  │  Euclidean measures                                    │
│    ╱   │  STRAIGHT LINE distance                                │
│   ╱    │                                                         │
│  ●─────┘  A [2,3]                                               │
│                                                                  │
│ Both direction AND magnitude matter                              │
│                                                                  │
│ Examples:                                                        │
│ [1, 1] vs [2, 2]  →  distance = 1.41 (different!)              │
│ [0, 0] vs [3, 4]  →  distance = 5.0                            │
│                                                                  │
│ Range: [0, ∞)                                                   │
│ Best for: Physical distances, when magnitude matters             │
└──────────────────────────────────────────────────────────────────┘

3.3 Dot Product
─────────────────────────────────────────────────────────────────────

Formula: dot_product = Σ(ai × bi)

Visual Representation:
┌─────────────────────────────────────────────────────────────────┐
│ Combines direction AND magnitude                                 │
│                                                                  │
│ [1, 1] · [1, 1] = 2                                             │
│ [2, 2] · [2, 2] = 8   (4x larger!)                              │
│                                                                  │
│ Special case: If vectors normalized (||v|| = 1)                 │
│               then dot_product = cosine_similarity              │
│                                                                  │
│ Range: [-∞, ∞]                                                  │
│ Best for: Collaborative filtering, pre-normalized vectors        │
└──────────────────────────────────────────────────────────────────┘

Metric Comparison Example:
┌─────────────────────────────────────────────────────────────────┐
│ Query: "action thriller" [0.9, 0.7, 0.1]                       │
│                                                                  │
│ Document 1: "action movie" [0.8, 0.6, 0.2]                     │
│   Cosine:    0.997 (very similar direction)                     │
│   Euclidean: 0.173 (close distance)                             │
│   Dot:       1.29  (high value)                                 │
│                                                                  │
│ Document 2: "romance film" [0.2, 0.3, 0.9]                     │
│   Cosine:    0.612 (different direction)                        │
│   Euclidean: 1.249 (far distance)                               │
│   Dot:       0.48  (low value)                                  │
│                                                                  │
│ All metrics agree: Doc1 > Doc2                                  │
└──────────────────────────────────────────────────────────────────┘

│                                                                  │

│                                                                  │

│                                                                  │
│ Query: "not a good movie"                                       │
│   Problem: Negation poorly captured, similar to "good movie"    │

│   Problem: Generic models don't understand specialized terms    │

Solution: Use Hybrid Search!
┌─────────────────────────────────────────────────────────────────┐

│       └────────────┬───────────────────┘                        │

│            & Combination                                         │
│                    ↓                                             │

│                     + (0.6 × semantic_score)                    │
│                    ↓                                             │
│            Unified Ranking                                       │
└──────────────────────────────────────────────────────────────────┘

SECTION 5: DENSE VS SPARSE EMBEDDINGS
═══════════════════════════════════════════════════════════════════════

5.1 Dense Embeddings (Traditional)
─────────────────────────────────────────────────────────────────────

Structure:
┌─────────────────────────────────────────────────────────────────┐
│ Text: "artificial intelligence"                                 │
│   ↓                                                              │
│ [0.234, -0.567, 0.891, 0.123, ..., -0.345]                     │
│  ↑       ↑       ↑       ↑            ↑                         │
│  All 768 dimensions have non-zero values                         │
│                                                                  │
│ Characteristics:                                                 │
│ • Dimensions: 384-1536                                           │
│ • Non-zero values: 100%                                          │
│ • Storage: 4 bytes × dimensions                                  │
│ • Every dimension is meaningful                                  │
│ • Black box (hard to interpret)                                  │
│                                                                  │
│ Storage (1M docs, 768d):                                        │
│   768 × 4 bytes × 1M = 3 GB                                     │
└──────────────────────────────────────────────────────────────────┘

5.2 Sparse Embeddings (Neural Sparse)
─────────────────────────────────────────────────────────────────────

Structure:
┌─────────────────────────────────────────────────────────────────┐
│ Text: "artificial intelligence"                                 │
│   ↓                                                              │
│ {                                                                │
│   "artificial": 0.85,     ← Explicit term                       │
│   "intelligence": 0.83,   ← Explicit term                       │
│   "ai": 0.72,             ← Learned synonym                     │
│   "smart": 0.68,          ← Expanded term                       │
│   "machine": 0.45,        ← Related concept                     │
│   "learning": 0.40        ← Related concept                     │
│ }                                                                │
│ Only ~250 non-zero out of 30,522 dimensions (~1%)              │
│                                                                  │
│ Characteristics:                                                 │
│ • Dimensions: 30,522 (BERT vocab)                               │
│ • Non-zero values: ~1-3%                                        │
│ • Storage: Only non-zero values                                  │
│ • Interpretable (can see which terms matter)                    │
│ • Uses inverted index (like BM25)                               │
│                                                                  │
│ Storage (1M docs):                                              │
│   ~250 terms × 8 bytes × 1M = 2 GB (smaller!)                  │
└──────────────────────────────────────────────────────────────────┘

Comparison Table:
┌─────────────┬──────────────┬──────────────┬──────────────┐
│ Aspect      │ Dense        │ Sparse       │ Winner       │

SECTION 6: QUANTIZATION
═══════════════════════════════════════════════════════════════════════

Memory Optimization Through Quantization
─────────────────────────────────────────────────────────────────────

Original (FP32):
┌─────────────────────────────────────────────────────────────────┐
│ 32 bits per value                                                │
│ [0.23456789, -0.56789012, 0.89123456, ...]                     │
│ Storage: 768 × 4 bytes = 3,072 bytes per vector                 │
│ Precision: ±1.2 × 10⁻⁷                                          │
│ Recall loss: 0%                                                  │
└──────────────────────────────────────────────────────────────────┘

FP16 (Half Precision):
┌─────────────────────────────────────────────────────────────────┐
│ 16 bits per value                                                │
│ [0.2346, -0.5679, 0.8912, ...]                                 │
│ Storage: 768 × 2 bytes = 1,536 bytes per vector                 │
│ Savings: 50% ✓                                                  │
│ Precision: ±5 × 10⁻⁴                                            │
│ Recall loss: <0.5% ✓                                            │
└──────────────────────────────────────────────────────────────────┘

INT8 (8-bit Integer):
┌─────────────────────────────────────────────────────────────────┐
│ 8 bits per value                                                 │
│ Scale float [-1.0, 1.0] to integer [-128, 127]                 │
│ [30, -72, 113, ...]                                             │
│ Storage: 768 × 1 byte = 768 bytes per vector                    │
│ Savings: 75% ✓✓                                                 │
│ Precision: ~0.008 per step                                       │
│ Recall loss: ~1% ✓                                              │
└──────────────────────────────────────────────────────────────────┘

Binary (1-bit):
┌─────────────────────────────────────────────────────────────────┐
│ 1 bit per value                                                  │
│ Rule: value ≥ 0 → 1, value < 0 → 0                             │
│ Original: [0.234, -0.567, 0.891, -0.123, ...]                  │
│ Binary:   [1, 0, 1, 0, ...]                                     │
│ Storage: 768 ÷ 8 = 96 bytes per vector                          │
│ Savings: 97%! ✓✓✓                                               │
│ Search: Fast initial search → Rescore with full precision       │
│ Recall loss: <2% with rescoring ✓✓                              │
└──────────────────────────────────────────────────────────────────┘

Visual Comparison:
┌─────────────────────────────────────────────────────────────────┐
│ Storage for 1M vectors (768 dimensions)                         │
│                                                                  │
│ FP32:    ████████████████████████████████ 3,072 MB             │
│                                                                  │
│ FP16:    ███████████████ 1,536 MB (50% off)                    │
│                                                                  │
│ INT8:    ████████ 768 MB (75% off)                              │
│                                                                  │
│ Binary:  █ 96 MB (97% off!)                                     │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘

💡 KEY CONCEPTS EXPLAINED

Vector Search in 60 Seconds:
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Traditional:  "apple headphones" → Exact words                │
│                ❌ Misses "wireless earbuds"                    │
│                                                                 │
│  Semantic:     "apple headphones" → Meaning                    │
│                ✅ Finds "AirPods", "earbuds", "wireless"      │
│                                                                 │
│  How: Text → Embedding Model → Vector                          │
│       [0.23, -0.45, 0.67, ...]                                 │
│       Find similar vectors → Return results                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

Three Search Methods:
┌──────────┬──────────────────┬────────────────────────────────┐
│ Method   │ How              │ When to Use                    │
├──────────┼──────────────────┼────────────────────────────────┤
│ Keyword  │ Exact term match │ SKUs, codes, precise terms     │
│ Semantic │ Meaning-based    │ Natural language, concepts     │
│ Hybrid   │ Combines both    │ Best for most use cases ✨     │
└──────────┴──────────────────┴────────────────────────────────┘

⚙️ CONFIGURATION EXAMPLES

Fast Setup (Low latency):
─────────────────────────
LOCAL_MODEL_NAME = 'all-MiniLM-L6-v2'  # 384 dimensions
HNSW_M = 8
HNSW_EF_CONSTRUCTION = 128
HNSW_EF_SEARCH = 50

Balanced (Recommended):
──────────────────────
LOCAL_MODEL_NAME = 'all-MiniLM-L6-v2'  # 384 dimensions
HNSW_M = 16
HNSW_EF_CONSTRUCTION = 256
HNSW_EF_SEARCH = 100

High Accuracy:
─────────────
LOCAL_MODEL_NAME = 'all-mpnet-base-v2'  # 768 dimensions
HNSW_M = 32
HNSW_EF_CONSTRUCTION = 512
HNSW_EF_SEARCH = 200

Production (AWS):
────────────────
BEDROCK_MODEL_ID = 'amazon.titan-embed-text-v1'  # 1536 dimensions
With quantization (FP16) for 50% memory savings

💰 COST ESTIMATES

Development:
  Local only: $0 (uses local models)
  With AWS:   ~$5-10/month (minimal testing)

Production (100K documents, 1,000 queries/day):
  Storage:    ~$0.12/month (5GB at $0.024/GB)
  Search:     ~$72/month (10 OCU-hours/day)
  Total:      ~$75/month

Optimization Savings:
  FP16 quantization:     50% memory → 50% cost
  Binary quantization:   97% memory → 97% cost
  Sparse encoding:       10x smaller index
  Disk-based:           32x memory reduction
